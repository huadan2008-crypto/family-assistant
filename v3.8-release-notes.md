## v3.8 · 2026-06-20

### 🐛 修复
- **固定支出提醒误判问题**：彻底移除"商家关键词在≥2个月出现过就判定为固定支出"的自动检测逻辑（曾误报 Uber、Trip.com、Alipay、JR东日本、Suica 等出行/旅行类消费）。
  - 改为**用户手动标记**：在「记一笔支出」弹窗新增「⏰ 加入月度提醒」勾选框，由你自己决定哪些是真正的固定支出（上网费、手机费、水电费等）。
  - 每月仍会自动检查清单里的项目当月是否已录入，未录入会在财务页顶部提醒；可点击提醒里的 ✕ 随时取消某一项。
  - 数据存储：`appState.recurringWatchlist`（新字段，不影响现有 `bills` 结构）。

### ✨ 新增
- **公司结算单导出**（财务页新增「📊 导出结算单」按钮）：
  - 可选参数：公司/账本（Green Ring / Guoqi Research / 个人 / 出租房45 / 出租房15 / 住宅共用 / 待定 / 全部）、周期（周/月/季度/自定义范围）、分类（不勾选=全部）、是否含明细列表。
  - 导出为 **CSV 文件**（带 UTF-8 BOM，Excel/Numbers/Google Sheets 直接打开不乱码），无需引入额外依赖库。
  - 文件内容：第一部分「分类汇总」（分类/笔数/金额合计/HST合计），第二部分（可选）「交易明细」（日期/商家/分类/金额/HST/账本/记账人/是否有收据）。
  - 自动记住你上次选择的参数（存浏览器 localStorage，键名 `fa_export_prefs`），下次打开默认带出。

### 🔒 未改动
- Google OAuth、Drive sync、现有 `appState.bills` 结构完全未变，向后兼容。
- 旧版本数据打开后 `recurringWatchlist` 会自动初始化为空数组，不会报错。
